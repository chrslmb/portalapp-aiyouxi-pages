# portalapp-aiyouxi-pages

## 项目简介

本项目用于归档和发布多个独立 HTML 页面。这些页面通常用于特定场景下的展示、工具或交互体验，不绑定任何具体域名或网站。

仓库内容以静态 HTML 文件为主，可直接通过浏览器打开，或部署到任意静态托管服务。

## 目录说明

```
.
├── pages/          # 存放各独立 HTML 页面文件
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/         # 公共资源（图片、样式、脚本等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名通常反映其用途或功能。
- `assets/`：存放页面共享的静态资源，便于统一管理。

## 页面归档说明

- 每个页面均为自包含的 HTML 文件，内部已引用所需资源。
- 页面之间无依赖关系，可单独使用。
- 如需引用外部资源（如 CDN 库），已在页面内显式声明。

## 维护说明

- 新增页面：在 `pages/` 目录下添加 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件，注意保持向后兼容。
- 删除页面：移除文件前请确认无其他页面依赖。
- 资源更新：修改 `assets/` 下文件后，需同步更新引用该资源的页面。

## 使用方式

1. 克隆本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件。
3. 或部署到任意静态文件服务器。
